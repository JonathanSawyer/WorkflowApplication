This is an indempotent workflow system. Only a single set of changes at a time.
  • Approved Deletes leave the record in owner table and set an archive flag (records are effectively excluded from all queries through Nhibenrate mapping) .
  • Create requests dont end up in owner table until they are approved.
  • Update requests dont modify the owner until they are approved.
  • BL\Workflow contains generic classes for an update, create and delete workflow.
  • BL UserWorkflow.cs and User.cs are example payloads
  • The Service Layer is injected into the Web Api Controllers.
  • I use Fluent NHibernate with SQLLite and specifically the discriminator as a factory to load the correct update, create and delete workflow.
  • UI written in Angular and Backend Exposed through WebApi.
Future Improvements.
  • Manage multiple users trying to change record (UI as workflow wont allow it but throws a nasty exception).
  • Row level (Upgrade Lock)locking of changes.
  • Within a business setting you wouldnt have a maker approving their own changes
